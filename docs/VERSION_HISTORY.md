# ORIEL Version History and Current Status

ORIEL uses one consolidated source repository. Historical milestone records are preserved under `versions/`; the root source tree is the active implementation.

| Version | Main milestone | Repository status |
|---|---|---|
| 0.1.0 | Initial console interpreter, CLI and early VS Code support | Historical milestone record |
| 0.2.0 | Lists, loops, files, JSON and project commands | Historical snapshot |
| 0.3.0 | Diagnostics, initial typing, package/LSP foundations | Historical snapshot |
| 0.4.0 | Modules, imports and initial API experiments | Historical snapshot |
| 0.5.0 | SQLite/database and OpenAPI experiments | Historical snapshot |
| 0.6.0 | Application services, validation and authentication utilities | Historical snapshot |
| 0.7.0 | Console tooling, bytecode experiment, REPL, docs and benchmarking | Historical baseline snapshot |
| 0.7.1 | Core audit, compiler services, structured diagnostics and multi-IDE foundation | Integrated into active source |
| 0.7.2 | Static type-system completion | Integrated into active source |
| 0.7.3 | Modules and standard library | Current active implementation |
| 0.7.4 | Package manager | Planned next release |
| 0.7.5 | Language server and VS Code intelligence | Planned |
| 0.8.0 | Compiler, IR, bytecode and VM | Planned |
| 0.8.1 | Debugger, profiling and optimization | Planned |
